Prompt Original do Usuário

Preciso criar a especificação completa do Core Banking da LBPAY (Instituição de Pagamento).

Contexto:
Somos uma IP (Instituição de Pagamento) regulada pelo Banco Central do Brasil, participante direto do SPI (Sistema de Pagamentos Instantâneos - PIX). Precisamos implementar um Core Banking completo que atenda todas as regulamentações BACEN.

Documentação Fornecida:
• Circular BACEN 3.978 (PLD/FT - Prevenção à Lavagem de Dinheiro)
• Resolução BACEN 80 (Instituições de Pagamento)
• Manual completo do PIX v8.3
• Diagramas de arquitetura de microsserviços
• Fluxos de integração com BACEN SPI

Funcionalidades Necessárias:
1. Cadastro de clientes (PF e PJ) com KYC completo
2. Abertura de contas (Conta Corrente e Conta Pagamento)
3. PIX completo (envio, recebimento, chaves, devolução, agendamento)
4. TED/TEF
5. Boletos (emissão e pagamento)
6. Cartões pré-pagos
7. Análise de risco e compliance (PLD/FT)
8. Contabilização (integração com TigerBeetle Ledger)
9. Reporting regulatório (COAF, BACEN)

Arquitetura Técnica:
• Microsserviços em Go e Python
• PostgreSQL para dados transacionais
• TigerBeetle para ledger contábil
• Redis para cache
• Kafka para eventos
• Kubernetes para orquestração
• Integração mTLS com BACEN SPI

Saída Esperada:
Gere a especificação técnica completa incluindo object_definitions, validation_rules, FSMs, integrações externas, e toda estrutura necessária para implementar este Core Banking usando a plataforma SuperCore.
